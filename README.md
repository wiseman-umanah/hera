# вҳ• HBAR Coffee Shop

> An AI barista on the Hedera testnet. Order coffee, pay on-chain with HBAR.

A full-stack Web3 demo where a conversational AI agent (Hera) takes your coffee order and processes a real Hedera testnet transaction вҖ” paid directly from your HashPack wallet.

**Built for the Hedera AI Agents Bounty вҖ” Week 1.**

---

## How it works

1. Visit the app and click **Connect Wallet** вҖ” your HashPack wallet connects via WalletConnect
2. Chat with **Hera**, the AI barista, to browse the menu and place an order
3. Confirm your order вҖ” Hera calls the Hedera Agent Kit to request payment
4. **HashPack pops up** for your approval вҖ” you sign the transaction
5. HBAR transfers on-chain, you get a receipt with a Hashscan link

Every payment is a real `TransferTransaction` on Hedera testnet. Nothing is simulated.

## Setup

### 1. Clone and install

```bash
git clone https://github.com/<your-username>/hbar-coffee-shop
cd hbar-coffee-shop

# Python backend
uv sync

# React frontend
cd frontend && pnpm install && cd ..
```

### 2. Configure environment

Copy `.env.example` to `.env` and fill in your values:

```bash
cp .env.example .env
```

```env
# Your Hedera testnet shop account (receives payments)
SHOP_ACCOUNT_ID=0.0.xxxxxx

# Groq API key вҖ” free at console.groq.com
GROQ_API_KEY=gsk_...

```


### 3. Fund your shop account

Get free testnet HBAR from the [Hedera Portal faucet](https://portal.hedera.com/dashboard).

## Architecture

```
Browser (HashPack)
    в”Ӯ
    в”Ӯ  WalletConnect вҖ” hedera_signAndExecuteTransaction
    в”Ӯ
React Frontend (Vite + @hashgraphonline/hashinal-wc)
    в”Ӯ
    в”Ӯ  WebSocket /ws
    в”Ӯ
FastAPI Backend (Python)
    в”Ӯ
    в”ңв”Җв”Җ LangGraph ReAct Agent вҖ” "Hera"
    в”Ӯ       в”ңв”Җв”Җ get_menu
    в”Ӯ       в”ңв”Җв”Җ calculate_order
    в”Ӯ       в”ңв”Җв”Җ check_balance  в”Җв”Җв–ә Hedera Mirror Node REST API
    в”Ӯ       в””в”Җв”Җ transfer_hbar_tool в”Җв”Җв–ә sends {type:"pay"} to frontend
    в”Ӯ
    в””в”Җв”Җ PaySession (asyncio.Event)
            в”Ӯ  waits for {type:"payment_done", tx_id}
            в–ј
        Agent receives TX ID вҶ’ outputs receipt with TRANSACTION_ID
```

**Payment flow in detail:**

1. Agent confirms order вҶ’ calls `transfer_hbar_tool`
2. Backend sends `{type: "pay", amount, from, to}` via WebSocket
3. Frontend builds a `TransferTransaction` using `@hashgraph/sdk`
4. `DAppSigner.call(tx)` sends it to HashPack via WalletConnect (`hedera_signAndExecuteTransaction`)
5. HashPack shows the approval popup вҖ” user approves in their wallet
6. HashPack signs, broadcasts on Hedera testnet, returns TX ID
7. Frontend sends `{type: "payment_done", tx_id}` back to backend
8. Backend resolves the payment вҶ’ agent outputs receipt + Hashscan link

## Utilities

### Derive private key from seed phrase

If you have a 24-word HashPack seed phrase and need the DER-encoded private key for your shop account:

```bash
uv run python derive_key.py
```

Tries both standard (`m/44'/3030'/0'/0'/0'`) and legacy Hedera derivation paths and auto-looks up your account ID on the testnet mirror node.
