# ☕ HBAR Coffee Shop

> An AI barista on the Hedera testnet. Order coffee, pay on-chain with HBAR.

A full-stack Web3 demo where a conversational AI agent (Hera) takes your coffee order and processes a real Hedera testnet transaction — paid directly from your HashPack wallet.

**Built for the Hedera AI Agents Bounty — Week 1.**

---

## How it works

1. Visit the app and click **Connect Wallet** — your HashPack wallet connects via WalletConnect
2. Chat with **Hera**, the AI barista, to browse the menu and place an order
3. Confirm your order — Hera calls the Hedera Agent Kit to request payment
4. **HashPack pops up** for your approval — you sign the transaction
5. HBAR transfers on-chain, you get a receipt with a Hashscan link

Every payment is a real `TransferTransaction` on Hedera testnet. Nothing is simulated.

## Setup

### 1. Clone and install

```bash
git clone https://github.com/<your-username>/hbar-coffee-shop
cd hbar-coffee-shop

# Python backend
uv sync

# React frontend
cd frontend && pnpm install && cd ..
```

### 2. Configure environment

Copy `.env.example` to `.env` and fill in your values:

```bash
cp .env.example .env
```

```env
# Your Hedera testnet shop account (receives payments)
SHOP_ACCOUNT_ID=0.0.xxxxxx

# Groq API key — free at console.groq.com
GROQ_API_KEY=gsk_...

# WalletConnect Project ID — free at cloud.walletconnect.com
WALLETCONNECT_PROJECT_ID=abc123...
```

> **Note:** `SHOP_PRIVATE_KEY` is optional. The shop account only **receives** HBAR — all payments are signed in the user's HashPack wallet, never server-side.

### 3. Fund your shop account

Get free testnet HBAR from the [Hedera Portal faucet](https://portal.hedera.com/dashboard).

## Architecture

```
Browser (HashPack)
    │
    │  WalletConnect — hedera_signAndExecuteTransaction
    │
React Frontend (Vite + @hashgraphonline/hashinal-wc)
    │
    │  WebSocket /ws
    │
FastAPI Backend (Python)
    │
    ├── LangGraph ReAct Agent — "Hera"
    │       ├── get_menu
    │       ├── calculate_order
    │       ├── check_balance  ──► Hedera Mirror Node REST API
    │       └── transfer_hbar_tool ──► sends {type:"pay"} to frontend
    │
    └── PaySession (asyncio.Event)
            │  waits for {type:"payment_done", tx_id}
            ▼
        Agent receives TX ID → outputs receipt with TRANSACTION_ID
```

**Payment flow in detail:**

1. Agent confirms order → calls `transfer_hbar_tool`
2. Backend sends `{type: "pay", amount, from, to}` via WebSocket
3. Frontend builds a `TransferTransaction` using `@hashgraph/sdk`
4. `DAppSigner.call(tx)` sends it to HashPack via WalletConnect (`hedera_signAndExecuteTransaction`)
5. HashPack shows the approval popup — user approves in their wallet
6. HashPack signs, broadcasts on Hedera testnet, returns TX ID
7. Frontend sends `{type: "payment_done", tx_id}` back to backend
8. Backend resolves the payment → agent outputs receipt + Hashscan link

## Utilities

### Derive private key from seed phrase

If you have a 24-word HashPack seed phrase and need the DER-encoded private key for your shop account:

```bash
uv run python derive_key.py
```

Tries both standard (`m/44'/3030'/0'/0'/0'`) and legacy Hedera derivation paths and auto-looks up your account ID on the testnet mirror node.
